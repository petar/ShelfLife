Integration

Integrating Sociability in your exisiting web infrastructure takes two steps. Since Sociability has its own API server for AJAX calls which also sets cookies, first you need to configure your server infrastructure to re-direct API requests to the Sociability server. Second, you will have to add a few lines to your existing web pages in order to enable the various features.

Configuring your server

The Sociability server serves HTTP API calls under the URL /api/ and some static resources (like style sheets and UI templates) under the URL /s/. The client-side JavaScript of Sociability (the script /s/js/ss.js) makes various AJAX calls to the API in order to perform its duties.

Let's say that your web pages are served under the hostname dpla.gov. For security considerations, browsers don't allow AJAX calls going to domains other than the domain where the web page came from (in this case dpla.gov). This is why, even though the Sociability server may run on a separate machine, your web server (perhaps something like Apache) or your front HTTP reverse proxy (perhaps something like NGINX) should be configured to forward all requests to URLS of the form dpla.gov/api/... and dpla.gov/s/... to the host that runs the Sociability server. Exactly how this is done depends on what software you are using.

Inserting sociability in your pages

In the head section: 

    <link href="/s/vanilla-style/stylesheets/sociability.css" rel="stylesheet" type="text/css" />
